## Technology Decisions & Rationale

### Why Next.js App Router?
- **Server Components**: Better performance
- **File-based routing**: Intuitive structure
- **Built-in API routes**: `/auth/callback`
- **React 18 features**: Streaming, Suspense

### Why Supabase?
- **All-in-one**: Auth + Database + Realtime
- **RLS built-in**: Secure by default
- **Real-time subscriptions**: WebSocket support
- **Free tier**: Perfect for development

### Why Tailwind CSS?
- **Utility-first**: Fast development
- **No CSS files**: Everything in components
- **Responsive**: Mobile-first by default
- **Small bundle**: Only used classes included

### Why Google OAuth?
- **User-friendly**: Everyone has a Google account
- **Secure**: No password storage needed
- **Profile data**: Avatar and name included
- **Quick setup**: Well-documented

- **Initial page load**: 1-2 seconds
- **Auth check**: <500ms (cached)
- **Fetch bookmarks**: <500ms (depends on count)
- **Add bookmark**: <1 second
- **Delete bookmark**: <500ms
- **Real-time update**: <1 second

### Scaling Considerations
- **Database**: PostgreSQL handles millions of rows
- **Realtime**: WebSocket connections scale horizontally
- **Vercel**: Auto-scales based on traffic
- **Supabase**: Free tier: 500MB DB, 2GB bandwidth

### Optimization Opportunities
1. Pagination for >100 bookmarks
2. Debounced search/filter
3. Image caching for avatars
4. Service Worker for offline support

## API Endpoints

### Supabase REST API
```
POST   /auth/v1/signup          # Create user (not used - Google OAuth)
POST   /auth/v1/token           # Exchange code for session
GET    /auth/v1/user            # Get current user
POST   /auth/v1/logout          # Sign out

GET    /rest/v1/bookmarks       # Fetch bookmarks (with RLS)
POST   /rest/v1/bookmarks       # Insert bookmark (with RLS)
DELETE /rest/v1/bookmarks?id=eq.{id}  # Delete bookmark (with RLS)
```

### Supabase Realtime
```
WS     /realtime/v1             # WebSocket connection
       Channel: bookmarks-changes
       Events: INSERT, UPDATE, DELETE
```

### Next.js Routes
```
GET    /                        # Home page
GET    /auth/callback           # OAuth callback handler
```

## Error Handling Strategy

### Client-Side Errors
```javascript
try {
  await supabase.from('bookmarks').insert(...)
} catch (error) {
  // User-friendly message
  setError('Failed to add bookmark. Please try again.')
  // Log to console for debugging
  console.error('Detailed error:', error)
}
```
